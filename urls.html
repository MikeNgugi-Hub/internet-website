Definition

A Uniform Resource Locator (URL) is an identifier for the location of a document on a website. URLs are the complete specifications of the locations of resources on the Internet. Every web page, image and file accessible on the Internet has its own unique URL that tells the browser exactly where to find it.

Structure of a URL

A basic URL looks like this:

http://www.awbc.com/info/regesstepp/index.html

A URL is made up of the following parts:

  • Protocol — specifies how the browser should retrieve the resource e.g. http://
  • Domain name — the address of the web server e.g. www.awbc.com
  • Top Level Domain (TLD) — the extension at the end of the domain e.g. .com
  • Path — the location of the specific file on the server e.g. /info/regesstepp/
  • Resource name — the specific file being requested e.g. index.html
  • Port number — port 80 is used for HTTP and port 443 is used for HTTPS

URLs, URIs and URNs

A URL is a specific kind of Uniform Resource Identifier (URI). A URI identifies a resource that can be downloaded from the web. The term URL is very widely used, but URI is the more general and technically correct term.

Another specific type of URI is the Uniform Resource Name (URN). The purpose of a URN is to provide a globally unique name for something, not necessarily to point to a web-based resource. Unlike a URL which tells you where something is, a URN simply gives it a unique name.

Example of a URL

A real-world example of a URL is:

http://www.library.strathmore.edu

In this example, http:// is the protocol, www.library.strathmore.edu is the domain name, and .edu is the top level domain indicating it is an educational institution.
