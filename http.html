Definition

HTTP stands for Hypertext Transfer Protocol. It is the protocol used by the World Wide Web to send and receive information between a web client and a web server. When using this protocol, the domain name or IP address is preceded by http:// in the address bar of the browser.

HTTP as a Request-Response Protocol

HTTP is a request-response protocol. This means that communication always begins with the client sending a request to the server, and the server responding back with the requested content. This cycle happens every time you load a web page.

The request-response cycle works as follows:

  • The user types a web address or clicks a link in the browser.
  • The browser sends an HTTP request to the appropriate web server.
  • The web server searches its system for the requested resource.
  • The server responds to the request by sending back the resource contents.
  • The browser receives the response and displays the page to the user.

Types of HTTP Requests

HTTP requests are always of a specific type. The most common types are:

  • GET — requests data from a specified resource, such as loading a web page.
  • POST — sends data to the server, such as submitting a form.
  • HEAD — same as GET but only retrieves the headers, not the body.
  • PUT — uploads or replaces a resource on the server.
  • DELETE — deletes a specified resource from the server.
  • CONNECT — establishes a tunnel to the server.
  • OPTIONS — describes the communication options available for the resource.
  • TRACE — performs a message loop-back test along the path to the target resource.

HTTP and the World Wide Web

HTTP is one of the most important technologies that the World Wide Web adds on top of the underlying Internet. Without HTTP, web browsers would have no standard way to request and receive web pages from servers. It is the foundation of all data communication on the Web and works hand in hand with HTML and URLs to deliver content to users around the world.
