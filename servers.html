Definition

A web server, sometimes called an HTTP server or application server, is a program that serves content using the HTTP protocol. This content is frequently in the form of HTML documents, images and other web resources, but can include any type of file. The content served by the web server can be pre-existing static content or content that is generated on the fly, known as dynamic content.

How Does a Web Server Work?

At its core, a web server serves static content to a web browser by loading a file from a disk and delivering it across the network to the user's web browser. This entire exchange is mediated by the browser and server communicating with each other using the Hypertext Transfer Protocol (HTTP).

Static vs Dynamic Content

Static content refers to files that already exist on the server and are delivered to the browser exactly as they are stored. Examples include plain HTML pages and images.

Dynamic content refers to content that is generated on the fly by the server based on user requests, input data or other conditions. This is what makes web applications interactive and personalised for each user.

Web Servers and Web Applications

A group of web pages that are related and logically connected is called a website. These web pages are stored on computers called web servers. When a website becomes interactive — responding to user input and generating personalised content — it becomes a web application. The web server is the foundation that makes all of this possible by handling incoming requests and delivering the appropriate responses back to the client.
